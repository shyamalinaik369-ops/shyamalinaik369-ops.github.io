01 · The Problem

Losing a customer costs 5×
more than keeping one.

Customer churn — when a client closes their account and leaves — is one of the most expensive problems in banking. Acquiring a new customer costs roughly five times more than retaining an existing one. Yet most banks only discover a customer has churned after it has already happened.

Machine learning offers the ability to identify at-risk customers before they leave, enabling targeted interventions: loyalty offers, fee waivers, personalised outreach. But building a model that actually works requires confronting the core challenge head-on: in a dataset where only 20% of customers churn, a model that predicts "no churn" for everyone will claim 80% accuracy — and be completely useless.

This project built a full ML pipeline from scratch — Logistic Regression, SVM, and Random Forest, all implemented in NumPy without scikit-learn — to navigate this trap and produce a model that genuinely identifies churners.

The Core Challenge

With an 80/20 class split, standard accuracy is a deceptive metric. The project's real objective was not to maximise accuracy, but to maximise recall on the churn class — because a missed churner costs the bank real money, while a false positive (a retained customer who wouldn't have churned anyway) costs only a small retention offer.

02 · The Data

10,000 customers.
One critical imbalance.

The dataset contains 10,000 bank customers across France, Germany and Spain, with 11 features covering demographics, financial behaviour, and engagement. No missing values or duplicates were found. The challenge was structural, not technical.

Churn Rate by Country and Gender — Key EDA Findings

* Germany stands out with a ~32% churn rate vs ~16% for France and Spain. Female customers churn at ~25% vs ~16% for males. Chi-square test confirmed gender–churn relationship is statistically significant (p ≈ 2.24×10⁻²⁶). ANOVA confirmed country does NOT significantly affect tenure (p = 0.925).

Feature Correlations with Churn — Ranked

* Age (0.29) and Active Member status (−0.16) are the strongest predictors. Low linear correlations do not mean a feature is useless — ML models capture non-linear interactions, which is why all features were retained for modeling. Feature engineering added Balance/Salary ratio and Age/Tenure ratio as derived signals.

03 · The Pipeline

Every preprocessing choice
was deliberate.

Rather than importing scikit-learn model classes, all algorithms were implemented in NumPy from scratch — implementing sigmoid functions, gradient descent, hinge loss, and Gini impurity by hand. This demonstrates genuine understanding of the mathematics behind the models, not just API familiarity.

01
Data Preprocessing
Outlier capping, encoding, feature engineering

Age values above 70 were capped at 70 (capping method) to improve distribution uniformity — the 70+ cohort represented a statistically minimal segment. Country and gender were label-encoded and one-hot encoded. Two interaction features were engineered from the data: Balance-to-Salary Ratio and Age-to-Tenure Ratio — features that capture financial stability and loyalty dynamics respectively. No missing values required imputation; the dataset was clean on arrival.

10,000 rows · 11 features 0 missing values 0 duplicates Age capped at 70 2 engineered features
02
SMOTE + Class Weights
Addressing 80/20 imbalance with synthetic oversampling

The churn column showed severe class imbalance — 7,963 retained vs 2,037 churned. Left unaddressed, any model would optimise for the majority class and predict "no churn" for virtually everyone. Two complementary strategies were applied: SMOTE (Synthetic Minority Oversampling Technique) generated synthetic churn samples by interpolating between nearest neighbours in feature space, balancing both classes to 7,963 each. Custom class weights applied a penalty multiplier to the minority class during gradient descent, forcing the model to treat each churner as more "expensive" to misclassify.

SMOTE: 7,963 → 7,963 per class Undersampling: 2,037 per class Class weight: inverse frequency 10-fold cross-validation
03
Logistic Regression — NumPy from scratch
Sigmoid + binary cross-entropy + gradient descent

Logistic regression was implemented entirely from scratch: sigmoid activation, negative log-likelihood loss, and gradient descent optimisation. Training ran for 1,000 epochs at learning rate 0.01. The cost curve confirmed steady convergence from 0.693 (epoch 0) to 0.494 (epoch 900). Min-max normalisation was applied to all features. The model was evaluated both without preprocessing (baseline) and with SMOTE + weighted classes — the comparison produced the project's central insight about the deceptiveness of raw accuracy.

LR = 0.01 · Epochs = 1,000 Cost: 0.693 → 0.494 From-scratch NumPy implementation 80/20 train-test split
04
SVM — NumPy from scratch
Hinge loss + gradient descent + K-fold cross-validation

A custom SVM was implemented using hinge loss optimisation and gradient descent with margin constraints. Without preprocessing, the model completely failed — AUC of 0.50, accuracy of 0.00, predicting nothing useful. The failure was diagnostic: it revealed the critical role of standardisation and class balancing for margin-based classifiers. After adding custom standardisation (zero mean, unit variance), SMOTE, and 5-fold cross-validation, the SVM recovered to an average AUC of 0.65 and accuracy of 75.24%. Each fold's ROC curve was plotted to assess consistency.

No preprocessing: AUC = 0.50 ✗ After preprocessing: AUC = 0.65 Avg accuracy: 75.24% 5-fold cross-validation Custom standardisation function
05
Random Forest — attempted from scratch
Bootstrap sampling + Gini impurity + majority voting

Random Forest was implemented from scratch: bootstrap sampling, Gini impurity calculation, recursive tree building with majority voting across 10 trees (max depth 5, min split size 10). The architecture is theoretically the strongest of the three — ensemble methods handle imbalanced data and non-linear interactions better than single models. However, without optimised C implementations or GPU acceleration, training on 10,000 rows exceeded 11 minutes — making hyperparameter tuning infeasible. This is reported honestly as a computational limitation, not a model failure. The implementation is correct; the hardware was the constraint.

Computationally infeasible without sklearn 10 trees · max_depth=5 Gini impurity implemented in NumPy Training > 11 min on 10,000 rows Expected best performer when optimised

The Insight

The model with lower accuracy (67%) is the better model. It catches 247 real churners that the "accurate" model misses entirely. In banking, each correctly identified churner represents a retention opportunity worth thousands in lifetime value. Accuracy measures how often you're right. Recall measures whether you're right about the things that matter.

💡
"Recall is more critical than precision in churn prediction. Missing a churner (false negative) costs the bank a real customer. Sending a retention offer to a non-churner (false positive) costs only a small incentive. The business cost of the two error types is not symmetric — and your choice of metric should reflect that asymmetry."
06 · Key Results

What the data revealed
about churner behaviour.

🎂 Age is the Strongest Predictor

Correlation of +0.29 with churn — the highest of any feature. Older customers churn at disproportionately higher rates. This suggests age-targeted retention strategies: loyalty rewards for customers over 45, proactive service reviews for long-standing customers approaching retirement.

🇩🇪 Germany's Anomalous Churn Rate

German customers churn at ~32% vs ~16% for France and Spain. Interestingly, ANOVA confirmed that country does not significantly affect tenure (p = 0.925) — meaning German customers aren't leaving sooner, they're leaving at higher rates across all tenure bands. The cause is likely product or service dissatisfaction specific to the German market.

👩 Female Customers Churn More

Female customers churn at 25.1% vs 16.5% for males. Chi-square test: p ≈ 2.24×10⁻²⁶ — this relationship is not random. It is one of the most statistically significant findings in the dataset. Gender-specific product features or communication strategies may be warranted, particularly in Germany where the gap is most pronounced.

📴 Inactive Members Leave More

Active member status correlates at −0.16 with churn — the second strongest signal. Inactive customers are significantly more likely to churn. This is actionable: re-engagement campaigns targeting inactive members (no app login, no transactions in 90 days) could be highly cost-effective.

Logistic Regression — Cost Reduction Over 1,000 Epochs (SMOTE + Weighted)

* Cost (binary cross-entropy) declines steadily from 0.693 at epoch 0 toward convergence near 0.66 by epoch 1,000. The smooth curve confirms a well-chosen learning rate of 0.01 — no oscillation, no plateau. This is the custom NumPy gradient descent implementation.

07 · Skills Demonstrated

What this project
actually proves.

🔧 ML from Scratch in NumPy

Implemented sigmoid function, binary cross-entropy loss, gradient descent, hinge loss, Gini impurity, and bootstrap sampling — all in raw NumPy without scikit-learn model classes. This demonstrates genuine understanding of ML mathematics, not just API familiarity.

⚖️ Imbalanced Data Mastery

Applied SMOTE, custom class weights, and undersampling to address 80/20 class imbalance. Made the principled argument that recall — not accuracy — is the right metric for churn prediction, and demonstrated it with a concrete comparison that any employer can immediately understand.

🧪 Statistical EDA

Applied chi-square test (gender vs churn: p ≈ 2.24×10⁻²⁶), ANOVA (country vs tenure: p = 0.925), correlation matrices, IQR outlier detection, and group-level analysis by country and gender. Statistical tests informed feature selection decisions rather than correlation cutoffs alone.

📊 ROC / AUC + K-Fold Validation

Generated ROC curves for each model and each cross-validation fold. Computed AUC as the primary discriminative metric. Applied 5-fold cross-validation to the SVM to assess generalisation — the consistent fold-level AUC (~0.61–0.67) confirmed no overfitting in the best implementation.
